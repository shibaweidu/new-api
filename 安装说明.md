## 第三方中转 API 支持

✅ **完全支持添加第三方中转 API 作为供应商**

### 一键获取模型列表

**是的！支持一键获取第三方的所有模型！**

#### 🚀 自动获取模型功能

在添加渠道时，New API 提供了自动获取模型列表的功能：

1. **添加渠道后**，在渠道编辑页面
2. 找到 **"获取模型列表"** 或 **"Fetch Models"** 按钮
3. 点击后系统会自动调用第三方 API 的 `/v1/models` 接口
4. 自动获取并导入所有可用模型
5. 你可以选择性启用需要的模型

#### 📋 支持的获取方式

**方式一：自动获取（推荐）**
- 适用于兼容 OpenAI API 格式的服务
- 自动调用 `/v1/models` 接口
- 一键导入所有模型

**方式二：手动配置**
- 适用于特殊格式的 API
- 手动输入模型名称列表
- 支持批量添加

**方式三：预设模板**
- 系统内置常见服务商的模型列表
- 选择服务商类型自动填充
- 包括 OpenAI、Claude、Gemini 等

#### 💡 使用步骤

```
1. 登录管理后台 → 渠道管理
2. 点击"添加渠道"
3. 填写基本信息：
   - 渠道类型：OpenAI
   - API 地址：https://api.third-party.com/v1
   - API Key：sk-xxxxx
4. 点击"获取模型列表"按钮
5. 系统自动获取并显示所有可用模型
6. 勾选需要启用的模型
7. 保存配置
```

#### ⚙️ 高级选项

- **模型映射**：可以重命名模型（如将 gpt-4 映射为 gpt-4-custom）
- **模型分组**：按用途或价格分组管理
- **批量启用/禁用**：快速管理大量模型
- **定时同步**：定期自动更新模型列表

#### 🎯 实际应用场景

**场景 1：聚合多个中转商**
```
渠道 A：获取 50 个模型
渠道 B：获取 30 个模型
渠道 C：获取 20 个模型
总计：100+ 个模型统一管理
```

**场景 2：成本优化**
- 同一个模型在不同渠道价格不同
- 自动选择最便宜的渠道
- 或按权重分配流量

**场景 3：高可用**
- 主渠道故障自动切换备用渠道
- 多渠道负载均衡
- 提高服务稳定性

### 支持的渠道类型

#### 1️⃣ OpenAI 兼容格式
- 任何兼容 OpenAI API 格式的第三方服务
- 支持自定义 API 地址
- 支持自定义模型列表
- 示例：国内各种 OpenAI 中转服务

#### 2️⃣ 官方 API 格式
- **OpenAI**: 官方 API 和 Azure OpenAI
- **Claude**: Anthropic Claude API
- **Google Gemini**: Google AI Studio
- **其他**: DeepSeek、Qwen、智谱等

#### 3️⃣ 自定义渠道
- 支持完全自定义 API 地址
- 可配置请求头和参数
- 灵活的格式转换

### 添加第三方供应商步骤

1. **登录管理后台**: http://localhost:3001
2. **进入渠道管理**: 点击左侧菜单"渠道"
3. **添加新渠道**: 点击"添加渠道"按钮
4. **配置渠道信息**:
   - 选择渠道类型（OpenAI、Claude、自定义等）
   - 填写 API 地址（第三方中转地址）
   - 填写 API Key
   - 选择支持的模型
   - 设置优先级和权重
5. **测试连接**: 点击测试按钮验证配置
6. **保存启用**: 保存后即可使用

### 高级配置

#### 🎯 智能路由
- **权重随机**: 多个渠道按权重分配请求
- **自动重试**: 失败自动切换到备用渠道
- **优先级**: 设置渠道优先级顺序

#### 💰 灵活计费
- 每个渠道可设置不同的价格倍率
- 支持缓存计费（降低成本）
- 按模型、按用户分别计费

#### 🔐 安全控制
- Token 级别的渠道访问控制
- 模型白名单/黑名单
- 速率限制
